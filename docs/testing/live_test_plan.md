# План live-проверки

Live-проверка выполняется на специально выбранном воркбуке и объекте.

1. Проверить `dl_runtime_status` и `dl_auth_probe`.
2. Прочитать воркбук, объект и его связи.
3. Выполнить plan-only и убедиться, что DataLens не изменился.
4. Выполнить save-only и проверить saved readback без публикации.
5. Выполнить обычное update и проверить save, saved readback, publish-from-saved и published readback.
6. Проверить видимый результат в DataLens.
7. Для отдельного тестового объекта проверить manifest action
   `retire_legacy_objects` с двухшаговым `confirm_delete`.
8. Убедиться, что логи и отчёты не содержат токенов или закрытых данных.

Производственные объекты для этой проверки не используются.
