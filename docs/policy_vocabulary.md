# Canonical Policy Vocabulary

These terms are the active vocabulary for README files, operator guides,
template `AGENTS.md` files, MCP response contracts, and docs consistency tests.
Generated validation evidence belongs in the ignored `artifacts/` workspace
and is not a source of public runtime policy.

| Term | Canonical meaning | Primary operator tools and artifacts |
| --- | --- | --- |
| `project-live` | A manifest-backed workflow for a real project repository with exact target IDs and declared commands. It is not a generic script runner. | `dl_detect_project_live_workflows`, `dl_plan_project_live_workflow`, `dl_run_project_live_dry_run`, `dl_run_project_live_apply`, `dl_read_project_live_summary`; manifest files `.datalens-mcp.json`, `datalens-mcp.project.json`, `.datalens-mcp.yaml`, `.datalens-mcp.yml`; schema `schemas/project-live-workflow-manifest.schema.json`. |
| `zero evidence` | A validation path that checked no declared files, SQL statements, readback entries, target IDs, or proof artifacts. Zero evidence reports fail with `zero_coverage` or an explicit blocked status. | `dl_validate_project`, `dl_read_project_live_summary`, `dl_build_validation_evidence_report`; artifacts listed by the project manifest or validation report. |
| `delivery-intent` | The save/publish state machine that classifies read-only, plan-only, save-only, save-then-publish, publish-from-saved, and blocked states. Intent is not permission. | `dl_build_payload_plan`, `dl_create_safe_apply_plan`, `dl_execute_safe_apply`, `dl_create_publish_from_saved_plan`, `dl_readback_and_report`; `docs/mcp/delivery_intent_policy.md`, `config/datalens_delivery_policy.json`. |
| `retire lifecycle` | The only allowed removal lane for named unnecessary DataLens objects. It is separate from normal validate, dry-run, save, publish, and readback workflows. | `action=retire_legacy_objects` in the project-live manifest; relation graph proof, saved no-reference proof, published no-reference proof, dry-run retire plan, approval provenance, execution summary, and post-retire readback. |
| `manifest summary` | A JSON artifact declared by a project-live manifest and parsed by MCP instead of trusting stdout. It records target IDs, changed counts, branch status, evidence paths, drift, dashboard preflight, and SQL lint coverage. | `dl_plan_project_live_workflow`, `dl_read_project_live_summary`; manifest `summary_path` and declared `evidence_checks`. |
| `hidden/internal compatibility tools` | Compatibility helpers used only by tests or explicit internal flows. They are not a user-selectable profile and are absent from the normal `tools/list` surface. | Normal surface: `tools/list`; test-only flags: `DATALENS_MCP_TEST_ONLY_REGISTRY=1` plus `DATALENS_MCP_ALLOW_HIDDEN_TOOL_CALLS=1`; diagnostics in `dl_runtime_status`. |
| `connector/connection` | Use `connector` for MCP planning routes and local requirements language; use `connection` for DataLens API object names, IDs, and readback payload fields. Do not treat the words as separate object families. | `dl_create_connector_plan`, `dl_update_connector_plan`, `dl_read_object`; DataLens `connectionId`; requirements connector plans. |
| `proof levels` | Explicit proof taxonomy attached to validation, safe-apply, readback, and deployment reports. A generic `ok` is not proof without context. | `proof_level`, `proof_levels`, or `ok_proof_context`; allowed values `source_static`, `installed_static`, `live_read_only_api`, `save_readback`, `publish_readback`, `browser_rendered`, `controlled_live_write`. |
| `current docs/API reconciliation` | The compact policy layer that compares current official DataLens docs/OpenAPI against local MCP support and route policy. It does not paste full docs into chat. | `dl_reference(mode='current_docs_delta')`, `dl_reference(mode='api_contract')`; `config/datalens_docs_feature_policy.json`, `config/datalens_api_operation_policy.json`, `docs/datalens/current_docs_reconciliation.md`, `docs/datalens/api_contract_coverage.md`. |
| `golden runtime gallery` | A generated regression fixture for supported, reference-only, unsupported, and banned route contracts. It is not a business dashboard and static checked-in contracts may have unavailable live/browser proof. | `scripts/build_golden_runtime_gallery.py`; `config/golden_runtime_gallery_inventory.json`, `config/golden_runtime_gallery_contracts.json`, `examples/golden_runtime_gallery/`; generated evidence under `artifacts/`. |
| `Wizard-first` | Deterministic create policy: preserve fresh saved technology on update; otherwise honor an explicit route, use a registered JavaScript capability gap, then default standard chart semantics to `wizard_native`. It is not a retry or fallback after transport failure. | `config/route_selection_policy_v5.json`, `docs/route-policy.md`, `src/datalens_dev_mcp/pipeline/route_registry.py`. |
| `ql_explicit` | QL read/create/update lane selected only after a direct user request. Create/update also require approval provenance plus an explicit payload or fresh saved QL seed. Automatic QL selection, prompt-to-QL generation, and QL delete are closed. | Generic lifecycle tools, `config/route_selection_policy_v5.json`, API method policy. |

## Active Docs Consistency Scope

Docs consistency checks scan active guides, README variants, and public MCP
contracts. Generated reports are ignored and cannot override those contracts.
